ScorllView和ListView嵌套显示不全解释

发表于 2016-03-16   |   分类于 Android   |  

在Android开发中。ScorllView嵌套ListView的情况很多,大家都知道怎么解决了,那为什么要这样做呢?

ScorllView嵌套ListView显示不全,因为listview的onMeasure不能正确计算出他的高度。这时候我们可以通过手动设置高度或重写onMeasure来实现。现在解决方案中最常用的和最好用的就是重写ListView的onMeasure函数来计算高度。代码如下：

@Override
protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {
    int expandSpec = MeasureSpec.makeMeasureSpec(Integer.MAX_VALUE >> 2, MeasureSpec.AT_MOST);
    super.onMeasure(widthMeasureSpec, expandSpec);
}

看代码很简单,主要一句就是int expandSpec = MeasureSpec.makeMeasureSpec(Integer.MAX_VALUE >> 2, MeasureSpec.AT_MOST);可是这句话的意思是什么呢?估计很多人都不知道。其中makeMeasureSpec是根据大小和模式返回一个int类型的值。在Android中一个控件所占的模式和大小是通过一个整数int来表示的。其中前面2位表示的是模式,后面30位表示的是大小。知道自定义View的开发者都知道MeasureSpec.AT_MOST的含义。这时候已经明白了。取int的最大值右移2位,获得后面的30位表示view最大的临界值。但是我们的实际情况又没那样大。所以采用了AT_MOST模式。
