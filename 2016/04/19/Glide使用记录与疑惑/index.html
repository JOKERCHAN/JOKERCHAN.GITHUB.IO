Glide使用记录与疑惑

发表于 2016-04-19   |   分类于 Android   |     |   阅读次数

前言

最近公司项目中,准备用一个新的图片加载框架。搜索了一段时间,决定用Google推荐的图片加载库Glide。
Glide的github地址为:Glide

在Android中使用

这里我们使用gradle进行依赖。

repositories {
    mavenCentral() // jcenter() works as well because it pulls from Maven Central
}

dependencies {
    compile 'com.github.bumptech.glide:glide:3.7.0'
    compile 'com.android.support:support-v4:23.1.0'
}

这样我们就可以在项目中使用了。当然你也可以用别的方式进行配置。

最简单的使用

// For a simple view:
@Override public void onCreate(Bundle savedInstanceState) {
  ...
  ImageView imageView = (ImageView) findViewById(R.id.my_image_view);

  Glide.with(this).load("http://goo.gl/gEgYUd").into(imageView);
}

混淆代码Proguard

-keep public class * implements com.bumptech.glide.module.GlideModule
-keep public enum com.bumptech.glide.load.resource.bitmap.ImageHeaderParser$** {
  **[] $VALUES;
  public *;
}

基本用法也就这样了。也没过多的去看别的。想的时候具体开始项目的时候在去看看。但是昨天吃饭的时候,提到了好像这个框架不支持设置缓存目录。我带着疑惑去查看了相关内容。其实是可以的。

自定义缓存目录

glide的硬盘缓存也是用的disklrucache所以还是应该去了解内部的原理,在使用开源库的时候。

首先实现一个自定义GlideModule然后用GlideBuilder去实现重新定义缓存路径,大小等。

/**
 * 自定义glidemodule
 * Created by JOKERCHAN on 2016/4/19.
 */
public class JGlideModule implements GlideModule {

    public void applyOptions(Context context, GlideBuilder builder) {

//                cacheLocation.mkdirs();
//                return DiskLruCacheWrapper.get(cacheLocation, 250 * 1024 * 1024);
//            }
//        });
        String cacheDir = "imgcache";
        //builder.setDiskCache(new ExternalCacheDiskCacheFactory(context, cacheDir, 250 * 1024 * 1024));
        builder.setDiskCache(new InternalCacheDiskCacheFactory(context, cacheDir, 250 * 1024 * 1024));

//        MemorySizeCalculator calculator = new MemorySizeCalculator(context);
//        int defaultMemoryCacheSize = calculator.getMemoryCacheSize();
//        int defaultBitmapPoolSize = calculator.getBitmapPoolSize();
//
//        int customMemoryCacheSize = (int) (1.2 * defaultMemoryCacheSize);
//        int customBitmapPoolSize = (int) (1.2 * defaultBitmapPoolSize);
//
//        builder.setMemoryCache( new LruResourceCache( customMemoryCacheSize ));
//        builder.setBitmapPool( new LruBitmapPool( customBitmapPoolSize ));
    }

    @Override
    public void registerComponents(Context context, Glide glide) {

    }
}

其中ExternalCacheDiskCacheFactory和InternalCacheDiskCacheFactory代表着一个是sdcard路径,另外一个则是自身路径。cacheDir代表的是路径下文件夹的名字。
ExternalCacheDiskCacheFactory代表/sdcard/Android/data/<application package>/cache
InternalCacheDiskCacheFactory代表/data/data/<application package>/cache

然后在manifest中进行设置

<meta-data
        android:name="com.jokerchan.baseglide.JGlideModule"
        android:value="GlideModule" />

其中name一项为你的GlideModule全路径

最后别忘记在设置混淆

-keepnames class * com.jokerchan.baseglide.JGlideModule

这样就按照自己的设置的路径进行缓存图片了。

集成网络库

通过HTTP/HTTPS从网络上下载图像并显示是非常重要的一块。虽然标准的Android网络包也能做这些工作，但在Android中开发了很多提升网络的模块。每个库有它自己的优势和劣势。最后，这其实需要项目的配合和开发人员自己的品位来决定的。

Glide 的开发者不强制设置网络库给你，所以 Glide 可以说和 HTTP/S 无关。理论上，它可以与任何的网络库实现，只要覆盖了基本的网络能力就行。
Glide网络模块默认用的是HttpUrlConnection，现在我们可以替换成okhttp的实现方式。
官方默认给出了okhttp和volley的方式。来看看okhttp的方式。

dependencies {
        compile 'com.github.bumptech.glide:glide:3.7.0'
        compile 'com.android.support:support-v4:23.1.0'
        compile 'com.github.bumptech.glide:okhttp-integration:1.4.0@aar'
        compile 'com.squareup.okhttp:okhttp:2.5.0'
}

Gradle 会自动合并必要的 GlideModule 到你的 Android.Manifest。Glide 会认可在 manifest 中的存在，然后使用 OkHttp 做到所有的网络连接。

然后在application或者activity中的oncreate中注册

Glide.get(this).register(GlideUrl.class,InputStream.class,new OkHttpUrlLoader.Factory(new OkHttpClient()));

别忘记混淆配置

-keep class com.bumptech.glide.integration.okhttp.OkHttpGlideModule

给自己的意见

项目中需要进行封装,避免以后换框架的时候需要改的地方太多。如果封装了只需要改封装的方法而不会影响到所有的代码。
